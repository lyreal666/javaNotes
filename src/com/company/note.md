/*
         * java整形有byte,short,int,long
         * 占内存字节大小分别为1,2,4,8
         * 4字节差不多20亿大小
         * 长整形后面跟着一个L或l
         * java为了实现跨平台,在不同平台各种整形占用字节大小都是一样的
         */

        /* 进制
        * 十六进制使用OX或ox开头 oxFFFF OXFFFF
        * 八进制使用0(零)前缀,很容易混淆,建议不要使用八进制常数 010(8)
        * java7开始可以使用ob或OB写二进制数,0b1001(9)
        * java7开始还可以为数字字面量加下划线,编译器会自动去除这些下划线
        * 1_0000_0000 就表示一个亿
         */

        /* 浮点型
        * java浮点型包括float,double两种
        * 遵循IEE754标准
        *
        * float 字面值有一个F或f后缀如3.14F
        * float 占用4字节
        * float 精度挚友6,7位
        *
        * double 默认字面值就是double类型
        * 占用8内存字节
        * 也可以在后面加d或D,但没必要
        * 精度达到15位
        *
        * 实际上很少情况会使用低精度的float
        * 除非存储空间是瓶颈
        *
        * 可以使用16进制表示浮点数
        * 例如:0.125即十进制的2的负三次方可以表示为0x1.0p-3
        * 16进制表示尾数时默认底数p为2
        *
        * IEE754标准要求能表示以下三种情况:
        * 正无穷 Double.INFINITY
        * 负无穷 Double.NEGATIVE_INFINITY
        * NaN(不是数字) Double.NaN
        *
        * 使用Double.isNaN判断一个数是不是NaN,所有NaN互不相等
        *
        * 所有使用IEE754标准的语言的通病
        * 浮点数计算有误差
        * 2.0-1.1=0.89999999999
        * 如果计算不允许有任何误差应该使用BigDecimal类
         */

        /* char
        * 可以表示十六进制的值\u0000到\Uffff
        * 其实就是一个字节大小
        * java的转义序列可以用在所有带引号的字符串或字符中
        * 但是\u可以出现在没有引号包裹的部分
        * 说白了就是jvm可以解析unicode字符
         */

        /* java的转义序列
        * 常用的有\n,\r,\b(退格),\t(制表符),\".\',\\
        * 单引号也要转义,我的天
         */
